# MyAgent 软件需求清单

> 本文档依据当前仓库已实现功能反向整理，供软著或与产品对齐使用；条目编号为需求标识，可按登记材料需要裁剪。

## 一、总则

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| R-G-001 | 软件运行于 Windows、macOS、Linux 等桌面环境（以 electron-builder 目标为准） | 必须 |
| R-G-002 | 单次安装或便携运行后可在本机离线保留会话与设置（不涉及强制云端账号体系） | 必须 |
| R-G-003 | 提供中文与英文两套界面文案并可切换 | 必须 |
| R-G-004 | 提供浅色 / 深色 / 跟随操作系统 三种主题观感 | 应该 |

## 二、会话与对话

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| R-CH-001 | 支持创建多个并行会话并在侧栏列出 | 必须 |
| R-CH-002 | 支持切换当前会话、重命名会话、删除会话（含确认） | 必须 |
| R-CH-003 | 支持在会话列表中按关键词检索过滤 | 应该 |
| R-CH-004 | 支持与模型进行多轮问答；用户消息与助手消息在界面中可追溯展示 | 必须 |
| R-CH-005 | 支持为单条助手消息导出或复制 Markdown 等格式的呈现（以实现为准） | 应该 |
| R-CH-006 | 支持将整段会话导出为 Markdown / HTML | 必须 |
| R-CH-008 | 启用流式时在连接或接口错误下**保留已输出片段**并在后续展示**错误说明**；用户在**尚未生成可见内容**时中止可**不保留空头助手气泡** | 应该 |

## 三、模型管理

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| R-M-001 | 支持配置多个命名模型条目，字段包含提供商、接口地址、密钥、模型名等基本参数 | 必须 |
| R-M-002 | 在单次发送前可由用户选择当前会话使用的模型 | 必须 |
| R-M-003 | 在接口能力匹配时支持流式输出与中止生成 | 应该 |
| R-M-004 | 支持与 OpenAI 兼容协议、Ollama、以及项目内适配的 Claude / Gemini 等路线的对接（以实现与密钥为准） | 必须 |

## 四、附件与文档解析

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| R-A-001 | 用户可在发送前附加图片及常见文档（如 xlsx、docx、md、txt） | 必须 |
| R-A-002 | 主进程将可读附件解析为可供模型阅读的文本或 Markdown 表格表示并附于用户消息上下文 | 必须 |
| R-A-004 | 对体积较大的 Office / 文本附件可提升 stat 上限并按**提取正文字数上限**截断且向模型侧注入**截断提示**，避免单次请求无限膨胀 | 应该 |

## 五、联网搜索

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| R-W-001 | 可配置启用或关闭「联网检索」总体能力 | 必须 |
| R-W-002 | 支持至少一种无需第三方 Key 的检索链路（如 DuckDuckGo 摘要路径）以及可选 Tavily / Brave（需 Key） | 必须 |
| R-W-003 | 仅在用户输入符合配置的触发策略时发起检索，避免每句必读网 | 应该 |
| R-W-004 | 会话级可对「本对话是否沿用联网全局开关」做覆盖（以实现为准） | 应该 |

## 六、工作区与本地化知识

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| R-K-001 | 用户可配置单一工作区根目录路径（本机文件夹） | 必须 |
| R-K-002 | 在根目录存在时按约定文件名（MYAGENT_KNOWLEDGE.md、knowledge.md、README.md 等顺序）节选内容注入上下文 | 应该 |
| R-K-003 | 用户可对本机向量索引执行「重建/建索引」，索引范围受项目内文件类型与体量限制约束 | 应该 |
| R-K-004 | 用户可单独配置向量嵌入所使用的服务商、基准 URL、API Key（若需要）、模型名 | 必须 |
| R-K-005 | 对已完成索引的工作区支持按语义相似度检索分块并按条数与总字数上限合并入系统提示（不混入用户常驻聊天记录） | 应该 |
| R-K-007 | 对已落盘指纹的索引支持**增量更新**：仅变更或未索引源文件重新分块与嵌入，未改文件可复用向量分块 | 应该 |

## 七、导出与 Markdown 呈现

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| R-E-001 | 助手消息使用 Markdown 渲染，支持 GitHub Flavored Markdown 表格等 | 必须 |
| R-E-003 | 助手输出中 **Markdown 围栏代码块**及经判定的**整块代码型回复**均应提供**一键复制**代码能力 | 应该 |

## 八、系统集成与隐私

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| R-P-001 | 主要状态与会话偏好持久化于本机 Electron `userData` 目录策略下（与安装包同源机一致） | 必须 |
| R-P-002 | 在设置页提供清空本地数据的入口并具备二次确认（以实现为准） | 应该 |
| R-P-004 | 单机仅保留**一个主进程实例**持有 `userData`，重复启动应将已有窗口置前 | 应该 |
| R-P-005 | 会话等 Zustand 文件持久化应具备**合并写盘防抖**并在退出前尽可能**冲刷**以降低数据丢失窗口 | 应该 |

## 九、扩展能力（选配）

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| R-X-001 | 支持通过配置的图像生成链路根据模型回应中的语法或工具约定生成配图 | 可以 |
| R-X-002 | 支持通过助手回复中的约定调用本机应用启动能力 | 可以 |
| R-X-003 | 新用户可分步向导介绍主要操作，向导可跳过 | 可以 |
