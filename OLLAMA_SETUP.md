### 🎯 如何使用

#### 1. 启动对话
- 点击"新建对话"按钮
- 在顶部模型选择器中选择任意本地模型
- 开始对话！

#### 2. 切换模型
- 在对话窗口顶部，点击模型下拉菜单
- 选择你想用的模型
- 即时切换，无需重启

#### 3. 推荐用法
- **快速问答**: Qwen3-VL 2B 或 Qwen2.5-VL 3B（速度快）
- **日常对话**: Qwen3-VL 8B 或 Qwen2.5-VL 7B（平衡）
- **复杂任务**: Gemma4 26B 或 Gemma4 31B（能力强）
- **图像理解**: Qwen3-VL 系列或 Qwen2.5-VL 系列

### 📝 添加新模型

如果你想添加其他 Ollama 模型：

1. **安装模型**:
   ```bash
   ollama pull llama3.2
   ```

2. **在应用中添加**:
   - 点击右下角设置图标 ⚙️
   - 点击"添加模型"
   - 填写信息：
     - 名称: `Llama3.2 (本地)`
     - 提供商: `Ollama`
     - API 地址: `http://localhost:11434`
     - API 密钥: (留空)
     - 模型名称: `llama3.2`
     - 勾选"本地模型"

### ⚠️ 注意事项

1. **Ollama 服务必须运行**
   - 检查命令: `ollama list`
   - 启动命令: `ollama serve`

2. **显存要求**
   - 大模型需要更多显存
   - 如果显存不足，Ollama 会自动使用系统内存（速度会变慢）

3. **首次加载**
   - 第一次使用某个模型时，可能需要几秒加载
   - 后续使用会很快

### 🔍 故障排查

#### 问题: 模型无法连接
**解决方案**:
```bash
# 检查 Ollama 是否运行
ollama list

# 重启 Ollama 服务
ollama serve

# 测试 API
curl http://localhost:11434/api/tags
```

#### 问题: 响应很慢
**解决方案**:
- 切换到较小的模型（2B 或 3B）
- 检查显存使用情况: `nvidia-smi` (Linux) 或 `Activity Monitor` (macOS)

#### 问题: 显存不足
**解决方案**:
- 关闭其他占用显存的应用
- 使用较小的模型
- Ollama 会自动使用系统内存（降级运行）
